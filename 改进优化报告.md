## 二、逐项改进详情

### 问题 1：Expo SDK 50 已严重过时 🔴

**原文档**：指定 Expo SDK 50（2024年1月发布），react-native 0.73.6，expo-router ~3.4.0

**影响**：
- npm registry 可能无法安装对应精确版本
- react-native 0.73.6 存在已知安全漏洞（CVE 编号多个）
- 与当前 Node v24 存在兼容风险
- 缺少最新 SDK 的性能优化和新特性

**改进**：
```
expo:           ~50.0.0 → ~52.0.0  (跨越 2 个大版本)
expo-router:     ~3.4.0 →  ~4.0.0
react-native:    0.73.6 →  0.76.9
react:           18.2.0 →  18.3.1
```

---

### 问题 2：缺少关键依赖 🟡

**原文档**：仅列出基础依赖，缺少 iOS 安全区和状态栏控制所需的标配库

**影响**：
- iPhone X 及以上机型的刘海屏/灵动岛区域内容被遮挡
- 状态栏无法跟随主题自动调整颜色
- 启动画面无法程序化隐藏

**改进**：新增 4 个标配依赖
```diff
+ react-native-safe-area-context: 4.14.0  (安全区适配)
+ expo-status-bar:               ~2.0.0   (状态栏控制)
+ expo-splash-screen:            ~0.29.0  (启动画面)
+ expo-constants:                ~17.0.0  (系统常量)
```

---

### 问题 3：图表库兼容性风险 🟡

**原文档**：使用 `react-native-chart-kit ^6.12.0`

**影响**：
- 该库最后主更新在 2023 年，对 SDK 52 无官方兼容验证
- 依赖链复杂（react-native-svg + 自身封装），版本冲突概率高
- 深色/浅色模式样式自定义困难

**改进**：**移除 react-native-chart-kit**，基于已有的 `react-native-svg` 手写两个图表组件

| 组件 | 实现 | 代码量 |
|------|------|--------|
| `PieStatChart.tsx` | `<Svg><Path>` 扇形弧线 + 图例 | ~130 行 |
| `BarStatChart.tsx` | `<Svg><Rect>` 柱形 + `<Line>` 坐标轴 | ~140 行 |

**优势**：
- 零额外依赖，仅使用已在白名单中的 react-native-svg
- 颜色、文字、背景完全受控，主题适配一行代码
- 支持横向滚动（多天数据）
- 不存在第三方库版本兼容问题

---

### 问题 4：无 ErrorBoundary 🟡

**原文档**：未提及任何异常兜底机制

**影响**：任意子组件 JS 异常 → 整个 App 白屏崩溃，用户无任何恢复手段

**改进**：新增 `src/components/ErrorBoundary.tsx`
- 包裹全部子路由（在 `app/_layout.tsx` 中集成）
- componentDidCatch 捕获异常
- 兜底 UI：⚠️ 图标 + "页面出现异常" + "重新加载" 按钮
- 开发模式下 console.error 输出完整堆栈

---

### 问题 5：无数据迁移策略 🟡

**原文档**：AsyncStorage 直接读写，无版本控制

**影响**：后续 App 更新时，若数据结构变更，旧数据可能导致读取崩溃或数据丢失

**改进**：`storageHelper.ts` 新增版本迁移机制

```
STORAGE_KEYS 新增：
  DATA_VERSION: '@deepseek/data_version'

读取流程：
  getStoredVersion()
  ├── null    → 首次使用 → 初始化默认数据 → write version=1
  ├── 1       → 当前版本 → 正常读取
  └── >1      → 未来版本 → 提示需升级 App，兼容读取
```

---

### 问题 6：UUID 库需原生桥接 🟡

**原文档**：使用 `react-native-uuid ^2.0.2`

**影响**：该库并非纯 JS 实现，增加 Expo Go 兼容不确定性

**改进**：替换为纯 JS 的 `uuid ^9.0.1`
```diff
- react-native-uuid: ^2.0.2
+ uuid:             ^9.0.1
+ @types/uuid:      ^9.0.8
```

---

### 问题 7-9：性能和体验优化 🟢

| # | 原文档 | 改进 |
|---|--------|------|
| 7 | FlatList 无性能优化 | 添加 `removeClippedSubviews`、`maxToRenderPerBatch=15`、`windowSize=10`、`keyExtractor` |
| 8 | 删除无二次确认 | 所有删除操作接入 `Alert.alert` 二次确认弹窗 |
| 9 | 无下拉刷新 | 日志列表添加 `RefreshControl` |

## 五、图表方案详解

### 为什么不使用 react-native-chart-kit

1. **时间线**：最后主更新在 2023 年 → SDK 52(2025) 未经验证
2. **依赖链**：依赖 react-native-svg 的旧版本 → 与 sdk 52 自带版本冲突
3. **自定义**：深色模式需要多层配置覆盖 → 原生 SVG 一行代码搞定

### 手写方案实现

```
PieStatChart（饼图）:
  - 输入: CategoryStat[] (categoryName, duration, color, percentage)
  - SVG: <Path d="M cx,cy L arcX1,arcY1 A r,r 0 largeArc,1 arcX2,arcY2 Z">
  - 图例: 水平排列的色块 + 名称 + 百分比

BarStatChart（柱状图）:
  - 输入: DailyStat[] (date, totalDuration)
  - SVG: <Rect x y width height rx="4">  柱形
  - SVG: <Line>  坐标轴 + 网格线
  - SVG: <Text>  数值标签 + 日期标签
  - 外层: <ScrollView horizontal> 支持多天横向滚动
```
